# Static Skills Distribution

Move skills from generated templates to static source files in `skills/` at the repo root. `init` copies from the bundled directory instead of assembling from in-memory strings and template files.

## Context

- Skills are currently generated by `buildGauntletSkillContent()` (for run/check) or read from `src/commands/skill-templates/` (for the other 5)
- This prevents distributing skills via Flowkay or as a Claude Code plugin — the canonical source doesn't exist as standalone files
- The goal: a `skills/` directory in the repo that is the single source of truth, bundled in the npm package, and copied to `.claude/skills/` by `init`

## Steps

### 1. Create `skills/` directory with static skill files

Create `skills/` at the repo root containing all 7 skill subdirectories:

- `gauntlet-run/` — SKILL.md (freeze the output of `buildGauntletSkillContent("run")`), extract-prompt.md, update-prompt.md
- `gauntlet-check/` — SKILL.md (freeze the output of `buildGauntletSkillContent("check")`)
- `gauntlet-push-pr/` — SKILL.md
- `gauntlet-fix-pr/` — SKILL.md
- `gauntlet-status/` — SKILL.md
- `gauntlet-help/` — SKILL.md + references/
- `gauntlet-setup/` — SKILL.md + references/

For run and check, generate the current output once and save it. For the other 5, move the template files into the proper directory structure with SKILL.md as the filename.

### 2. Delete `src/commands/skill-templates/`

Remove the entire directory (15 files). All content now lives in `skills/`.

### 3. Rewrite skill installation in `init.ts`

- Delete `readSkillTemplate()`, `buildGauntletSkillContent()`, all skill content constants (`GAUNTLET_RUN_SKILL_CONTENT`, `PUSH_PR_SKILL_CONTENT`, `HELP_SKILL_BUNDLE`, etc.), and the `SKILL_DEFINITIONS` array
- Replace with a `SKILL_ACTIONS` list of the 7 action name strings
- Resolve the bundled `skills/` path relative to the compiled output (navigate from `dist/commands/` up to repo/package root)
- Rewrite `installSkillsWithChecksums()` to:
  - For each action, use `computeSkillChecksum()` on both the source (`skills/gauntlet-{action}`) and target (`.claude/skills/gauntlet-{action}`) directories
  - If target doesn't exist → recursive copy
  - If checksums match → skip
  - If checksums differ → prompt user, then recursive copy
- Delete `writeSkillFiles()` and `writeFilesToDir()` — replace with a generic recursive directory copy utility
- Remove `skillsOnly` field handling

### 4. Simplify `init-checksums.ts`

- Delete `computeExpectedSkillChecksum()` and its `toEntries()` helper — no longer needed since both sides are now real directories compared with `computeSkillChecksum()`
- Keep `computeSkillChecksum()`, `collectFiles()`, and all hook-related checksum functions unchanged

### 5. Update `package.json`

Add `"skills"` to the `files` array so the directory is included in the npm package.

### 6. Verify

- Run `npm run build` to confirm compilation
- Run existing tests
- Run `agent-gauntlet init` in a test project to confirm skills are copied correctly
- Verify the installed skills match the source files (checksums should be equal)
